# 동아시아·한반도 기후인자 기반 객관지수예보 구현 계획

업데이트 기준일: 2026-04-15

## 1. 문서 목적

이 문서는 현재까지 반영된 구현 결과를 기준으로, 한반도 기온·강수 3개월 전망(M+1, M+2, M+3)을 위한 기후인자 기반 별도 보고서 체계를 최신 상태로 정리한 구현 계획서다.

이번 업데이트에서 특히 반영한 사항은 다음과 같다.

- 남한 월별 실황 관측자료를 참값 기준으로 확정했다.
- 실황을 `-, 0, +` 기호 체계로 변환하는 경로를 구현했다.
- NOAA PSL `climateindices`와 `TimeSeries1` 전체 목록을 기준으로 인덱스 수집 범위를 재점검했다.
- `list/`만이 아니라 `timeseries/month` 메타데이터까지 활용해 누락 인덱스를 보강했다.
- 현재 어떤 자료가 확보되었고, 무엇이 아직 미완인지 운영 관점으로 다시 정리했다.

이 문서의 최종 목표는 다음과 같다.

- 기후인자 전용 별도 슬라이드 보고서를 안정적으로 생성한다.
- 각 기후인자의 최신 현황과 한반도 영향 부호(`-, 0, +`)를 월별로 정리한다.
- 실황 관측 참값을 이용해 3분위 확률을 최대한 객관적으로 산출한다.
- 보고서 안에 적절한 수의 기후인자 시계열을 넣고, 판단 근거를 추적 가능하게 남긴다.

## 3. 현재 확보된 데이터 자산

### 3.1 한반도 실황 참값

관측 참값은 사용자가 제공한 남한 월별 자료를 기준으로 한다.

- 평균기온 편차: `1973 ~ 2026`, 최신 가용 월 `2026-03`
- 강수량: `1973 ~ 2026`, 최신 가용 월 `2026-03`
- 관측 지점 수:
  - `1973 ~ 1989`: 56개 지점
  - `1990 ~ 현재`: 62개 지점

현재 생성된 참값 관련 산출물은 다음과 같다.

- `inputs/observed_symbols/남한_월별_실황_기호화.md`
- 값 표기 형식:
  - 기온: `2.3(+)`
  - 강수: `123.2(+)`

실황 기호 판정 원칙은 다음과 같다.

- 기온: 각 월의 `비슷범위 ±값`을 기준으로 `-, 0, +` 판정
- 강수: 각 월의 `비슷범위 하한~상한`을 기준으로 `-, 0, +` 판정

이 자료는 앞으로 모든 객관지수예보의 참값 기준으로 사용한다.

### 3.2 NOAA PSL 기후인자 수집 자산

현재 수집기는 다음 3개 축을 함께 사용한다.

- `https://psl.noaa.gov/data/climateindices/`
- `https://psl.noaa.gov/data/climateindices/list/`
- `https://psl.noaa.gov/data/timeseries/month/`

### 3.3 PSL TimeSeries1 전수 확인 결과

`TimeSeries1` 기준으로 다음 사항을 확인했다.

- `SOI`부터 `Globally integrated angular momentum`까지 전체 옵션을 직접 대조했다.
- 기존 `list/` 기반 수집만으로는 빠지던 항목을 보강했다.
- 새로 안정적으로 포함된 대표 항목:
  - `GBI`
  - `TNH`
  - `PMM`
  - `AO 20CR`
  - `AAO 20CR`
  - `NH Sea-Ice Extent`
  - `SH Sea-Ice Extent`
  - `ACE Atlantic`
  - `ACE Eastern Pacific`
  - `MDR Hurricanes SST Anomaly`
  - `TPI(IPO)` 수리 복구

현재 예외는 2건이다.

- `Monthly Hurricane Total`
  - PSL 옵션은 있으나 현재 raw가 월별 시계열로 파싱 가능한 형태를 제공하지 않아 `unparsed`
- `MDR - Tropics SST Anomaly`
  - PSL 옵션은 있으나 현재 공식 raw URL을 찾지 못해 `missing_source`

## 4. 데이터 해석 원칙

### 4.1 모든 인덱스를 같은 무게로 쓰지 않는다

PSL 저장소는 운영용 지수, 연구·참조용 지수, 업데이트가 멈춘 지수, 동아시아와 물리적 연결이 약한 지수를 함께 담고 있다. 따라서 전체 수집은 필요하지만, 해석과 확률 산출에는 중요도 차등을 둔다.

운영 원칙은 다음과 같다.

- `동아시아 직접성`이 강한 인자를 우선 사용
- `최신성`이 떨어지는 인자는 배경장으로만 사용
- `중복 의미`의 ENSO 계열은 묶어서 해석
- `운영용 최신 진단`과 `역사적 통계 연결`을 분리해서 사용

### 4.2 최신성 등급

현재 최신성 분류는 다음을 기준으로 유지한다.

- `fresh`: 기준월 대비 1개월 이내
- `usable_with_lag`: 기준월 대비 2~4개월 지연
- `stale`: 기준월 대비 5~12개월 지연
- `historical_only`: 12개월 초과 지연 또는 사실상 운영 해석 부적합

### 4.3 실황 참값 중심의 객관화 원칙

객관지수예보에서는 관측 참값과 기후인자를 직접 연결한다.

- 목표 변수:
  - 한반도 월평균기온 편차의 `-, 0, +`
  - 한반도 월강수량의 `-, 0, +`
- 설명 변수:
  - 각 기후인자의 월별 상태 또는 위상
- 리드:
  - 기준월 `M`에서 `M+1`, `M+2`, `M+3`

예시 표현은 다음과 같다.

- `PDO`: 기온 영향 `M+1 +`, `M+2 0`, `M+3 -`
- `PDO`: 강수 영향 `M+1 -`, `M+2 0`, `M+3 +`

이 부호 행렬을 바탕으로 3분위 확률을 계산한다.

## 단계 5. 3분위 확률 엔진 고도화

### 상태

핵심 미완 단계

### 구현 목표

기후인자 영향 부호와 과거 조건부 빈도를 이용해 한반도 기온·강수 3분위 확률을 산출한다.

### 기본 설계

1. 기준월 현재 각 인자의 최신 상태를 읽는다.
2. 각 인자에 대해 `M+1/M+2/M+3` 영향 부호를 조회한다.
3. 관련 표본의 과거 조건부 3분위 빈도를 계산한다.
4. 인자 중요도와 신뢰도를 반영해 결합한다.
5. 표본 부족 시 평년 33.3%로 수축한다.

### 결합 규칙 초안

- 핵심 인자군은 높은 가중치
- 배경 인자군은 보조 가중치
- stale/historical_only 인자는 제한적 사용
- 문헌과 최근 운영 진단이 충돌하면 강제 단정 대신 확률 폭을 넓힘

### 남겨야 할 설명 정보

- 어떤 인자가 확률을 올렸는지
- 어떤 인자가 상쇄했는지
- 표본 수가 충분한지
- 최신성 제약이 있는지

- 인자별 시계열 패널
- 인자별 한반도 영향 부호 표
- 기온/강수 3분위 확률 표 또는 패널
- 근거 문헌 및 기관 진단
- 사용 제외 또는 신뢰도 낮은 인자 명시

### 표현 규칙

- 부호는 `-, 0, +`로 명시
- 확률은 상·중·하 3분위로 명시
- 기준일과 최신 자료월을 절대 날짜로 명시
- stale 자료는 도표 또는 본문에 명확히 경고

## 8. 우선순위별 다음 작업 목록

### 우선순위 1

- 관측 참값과 기후인자를 연결하는 `lead-lag 영향 부호 행렬` 생성
- 핵심 인자군부터 `M+1/M+2/M+3` 기온·강수 영향 부호 자동화

### 우선순위 2

- 영향 부호를 3분위 확률로 변환하는 객관 확률 엔진 구현
- 표본 수 부족 시 shrinkage 규칙 구현

### 우선순위 3

- 슬라이드 보고서에 인자별 시계열과 영향 부호 표를 정식 반영
- 보고서 문구를 “기관 근거 + 객관 통계” 형식으로 정리

### 우선순위 4

- `Monthly Hurricane Total`과 `MDR - Tropics SST Anomaly` 재확인
- 전체 인자 중요도 테이블 정식화

## 9. 성공 기준

다음 조건을 만족하면 구현이 실질적으로 완료된 것으로 본다.

- 기준월 실행 시 최신 PSL/보완 자료를 자동 반영할 수 있다.
- 남한 실황 참값이 연도·월 단위로 `-, 0, +`로 재사용 가능하다.
- 핵심 기후인자에 대해 `M+1/M+2/M+3` 영향 부호가 자동 산출된다.
- 한반도 기온·강수의 3분위 확률이 재현 가능하게 계산된다.
- 별도 슬라이드 보고서 안에서 시계열, 영향 부호, 확률, 근거가 한 번에 제시된다.

## 10. 현재 리스크 및 주의사항

- PSL 저장소는 인덱스별 최신성이 서로 다르다.
- 모든 인덱스를 같은 무게로 쓰면 오판 가능성이 있다.
- 일부 인덱스는 운영용이 아니라 연구·참조용이다.
- 한반도 여름 전망은 ENSO 단일 지수보다 서북태평양 대류 및 중위도 파동 결합 영향을 크게 받는다.
- 문헌 기반 메커니즘과 역사적 조건부 통계가 완전히 같은 방향을 보장하지 않을 수 있다.

## 11. 운영 메모

현재 기준으로 이 프로젝트는 “자료 수집과 기본 보고서 생성” 단계는 갖춰졌다. 다음 개발의 중심은 “실황 참값과 기후인자를 연결한 객관 확률화”이며, 앞으로의 구현은 이 연결 고리를 만드는 쪽에 집중해야 한다.

즉, 이제부터의 핵심 개발 질문은 다음 하나로 요약된다.

`기후인자의 현재 상태를 과거 한반도 실황과 어떻게 연결해 M+1, M+2, M+3의 기온·강수 3분위 확률로 변환할 것인가?`

이 문서는 앞으로 그 작업을 기준으로 계속 업데이트한다.
